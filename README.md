# Stipuha

Локальный финансовый трекер для Obsidian. Главный показатель — сколько можно потратить до конца месяца с учётом балансов, ожидаемых доходов и будущих регулярных платежей.

## Установка в Obsidian

1. Откройте папку нужного vault и создайте в ней `.obsidian/plugins/financial-space/`.
2. Скопируйте в неё `main.js`, `manifest.json` и `styles.css` из этого проекта.
3. В Obsidian откройте **Settings → Community plugins**, отключите Restricted mode при необходимости и включите **Финансовое пространство**.

Для разработки после изменений выполните `npm run build`, затем перезагрузите Obsidian.

## Как считается прогноз

`доступно = балансы счетов + ожидаемые доходы − регулярные платежи, которые ещё впереди в этом месяце`

`дневной лимит = доступно / число дней до конца месяца`

Расход и доход меняют баланс выбранного счёта и сразу обновляют панель.

## Где лежат данные

Основные данные всегда хранятся в стандартном для Obsidian файле плагина:

`.obsidian/plugins/financial-space/data.json`

Он содержит счета, бюджеты, регулярные платежи и историю операций. Плагин не удаляет и не заменяет его Markdown-настройками.

Параллельно каждая новая операция создаёт обычную Markdown-заметку. По умолчанию это:

- `Операции/ГГГГ-ММ/*.md` — отдельная Markdown-заметка для каждой операции.

Путь к журналу и шаблон имён файлов настраиваются в **Settings → Финансовое пространство → Markdown-журнал операций**. Для пути доступна переменная `{month}`, а для имени файла — `{date}`, `{month}`, `{kind}`, `{category}`, `{account}`, `{amount}` и `{id}`.

Так операции участвуют в обычном поиске Obsidian и синхронизируются вместе с vault. `data.json` остаётся источником данных для панели; Markdown-файлы — удобным параллельным журналом.
